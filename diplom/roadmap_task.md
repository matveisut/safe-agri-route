**Тема:** «Разработка платформы автоматизированного планирования безопасных маршрутов для групп сельскохозяйственных дронов с учётом киберугроз»  
**Сроки:** 01.04.2026 – 01.06.2026 | **Защита:** 24.06.2026  
**Студент 1:** Сутормин М.П. (серверная часть, алгоритмы) | **Студент 2:** Яковенко В.С. (frontend, ИБ)

## Согласование с репозиторием `safe-agri-route` и документами `ТЗ.md` / `План обдуманный.md` (актуализация 19.04.2026)

Ниже — **фактическое состояние**, чтобы дорожная карта не противоречила коду и ТЗ.

**Уже реализовано в коде (MVP):** FastAPI; PostGIS; модули `risk_map.py`, `routing_service.py` (Risk-Weighted Voronoi + OR-Tools), `replanner.py`, `mavlink_service.py`; REST под префиксом **`/api/v1/mission/...`** (планирование, поля, зоны, старт, перепланирование); JWT (`/auth/login` и защита эндпоинтов); WebSocket **`/ws/telemetry`**; фронтенд (React/Vite, Leaflet, рисование полей/зон, RiskOverlay, метрики IRM, демо перепланирования); `docker-compose` (+ `docker-compose.sitl.yml`).

**Вне объёма MVP / не в репозитории:** Gazebo, ROS2 как обязательный стек; отдельный TLS/mTLS MAVLink-proxy и HMAC команд — в ТЗ для защиты достаточно **JWT + pymavlink**; полноценный audit log API — опционально для раздела 2; **автоматический** мультисенсорный контур (GNSS/IMU/RSSI/рой/SDR → fusion → карта риска) — см. **`ТЗ.md` §10**, в коде не реализован.

**Осталось для научной части (раздел 1.5 ВКР):** автономные скрипты **`backend/simulation/`** (baseline, runner, visualizer, графики) — см. `Промпты.md`, Промпт 8.

**Интерфейсы API в текстах работ:** не использовать устаревшие пути вида `GET /api/v1/fields` без префикса `mission` — см. `ТЗ.md` v1.2.
